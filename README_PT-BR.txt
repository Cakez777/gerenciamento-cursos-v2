Este projeto é uma aplicação fullstack desenvolvida para gerenciar cursos, alunos e matrículas, permitindo também a geração de relatórios. A proposta foi construir uma plataforma simples e funcional que centralizasse essas informações de maneira organizada e intuitiva. Para isso, utilizei Laravel no backend e React.js no frontend, ambos conectados por meio de uma API REST.

O backend foi desenvolvido em Laravel e é responsável por toda a lógica de negócio, autenticação e integração com o banco de dados. Para facilitar a configuração local, utilizei o XAMPP, que fornece o servidor Apache e o MySQL já integrados. Dessa forma, foi possível gerenciar o banco de dados de forma prática e manter a aplicação rodando sem dificuldades. O backend oferece endpoints para realizar operações de cadastro, listagem, edição e exclusão de alunos, cursos e matrículas, além de gerar relatórios com informações de desempenho, como faturamento e cursos mais acessados.

O banco de dados utilizado neste projeto pode ser criado a partir do arquivo SQL fornecido. Ele está localizado na pasta database/.sql dentro do repositório, com o nome ranhentos_db.sql. Para configurar o banco de dados localmente, basta abrir o phpMyAdmin pelo XAMPP, criar um novo banco com o nome desejado (por exemplo, ranhentos_db) e importar o arquivo SQL. Isso garantirá que todas as tabelas e dados necessários para a aplicação já estejam presentes, evitando a necessidade de criar o banco do zero.

Já o frontend foi implementado em React.js e construído para ser direto e amigável. Ele conta com uma página inicial que centraliza as opções do sistema e redireciona o usuário para as seções específicas, como cadastros e relatórios. Toda a comunicação com o backend é feita através de requisições à API, garantindo a integração entre as duas camadas.

Para rodar o projeto localmente, é necessário ter o XAMPP configurado e em execução, com o Apache e o MySQL ativos. Após clonar o repositório, dentro da pasta do backend é preciso instalar as dependências com o Composer, configurar o arquivo .env do Laravel (incluindo os dados do banco de dados importado), rodar as migrações caso necessário e executar php artisan serve para inicializar o servidor. No frontend, basta entrar na pasta correspondente, rodar npm install e em seguida npm start para iniciar o servidor React, que ficará disponível em http://localhost:3000.

Com isso, a aplicação fica totalmente funcional: o backend gerencia os dados com segurança e consistência, enquanto o frontend fornece a interface visual para interação do usuário. Este projeto foi pensado como um estudo prático para consolidar conhecimentos em Laravel, React e integração de sistemas, além de demonstrar como utilizar ferramentas como o XAMPP e arquivos SQL para acelerar o desenvolvimento local.
